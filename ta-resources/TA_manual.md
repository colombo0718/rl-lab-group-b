### 任務完成記錄表——怎麼用

- 每位學生完成任務後**舉手**，你過去確認畫面
- 確認通過 → 填入**完成時間**（如 `15:32`）
- 未完成 → 填 `✗`；課後補完 → 改填 `補`
- T5 Fighter（A 組）/ T5 延伸任務（B 組）為選做，有嘗試即記錄，**不納入完成率統計**
- 課後計算各任務完成率，填入表格底部的統計摘要

### 課堂觀察記錄表——怎麼用

- 每堂課一份，照時間欄位記錄「實際情況」（準時、延遲幾分鐘、順利與否）
- 學生提問、卡關、有趣觀察 → 隨手記在「二、學生互動觀察」
- 技術問題（平台掛掉、載入失敗）→ 記在「四、技術與操作問題」
- 課後整理完填給研究者，不需要寫成報告，重點是**具體、有細節**

## 三、自由體驗期間——你要觀察什麼

每段影片結束後學生自己操作。這段時間：

| 觀察項目 | 怎麼判斷 | 你該做什麼 |
|---|---|---|
| 卡住沒在動 | 超過 2 分鐘沒有操作 | 主動詢問 |
| 找不到圖表 | 盯著遊戲畫面、沒開 Analysis tab | 提示：右側 Analysis → 對應圖表 |
| 超參數沒改 | ε 還是預設值，沒跑任何 episode | 提示先照影片設定跑起來 |
| 提早做完 | 已完成任務在等 | 鼓勵多跑幾次、嘗試不同參數值 |
| 平台 bug | 畫面空白、遊戲不動 | 請學生重整；若多人同時發生請記錄並回報 |

## 四、各任務驗收方式與關鍵檢核點

驗收方式：學生**指著自己的螢幕**向你口頭說明，不用書面作答。  
通過後在任務完成記錄表填入時間。

---

### T1 · MAB — ε 探索 vs 利用

**任務：** 分別跑 ε=0.9 和 ε=0.1，描述 reward curve 差異。

**關鍵檢核點：**

| 等級 | 學生說了什麼 |
|---|---|
| ✅ 通過 | ε=0.9 前期波動大、探索多；ε=0.1 較快收斂但容易卡在次優解 |
| ✅ 通過 | 大 ε 比較「亂」，小 ε 比較「穩定但不一定最好」（概念對） |
| ⚠️ 追問 | 只說「一個高一個低」但說不出原因 → 問：為什麼？ |
| ❌ 重做 | 沒有跑兩個 ε、或說不出任何差異 |

---

### T2a · Maze 1D — 指出 SAR / Episode

**任務：** 在 Maze 1D 遊戲裡，指出 State / Action / Reward / Episode 各是什麼。

| 元素 | 正確答案 | 常見錯誤 |
|---|---|---|
| State | Agent 目前在哪一格（位置） | 說「分數」或「整個畫面」 |
| Action | 向左 / 向右 | 說「按按鈕」 |
| Reward | 到終點 +1、踩雷 −1、其他 0 | 只說「得分」 |
| Episode | 從出發到結束（到終點或踩雷）為一局 | 說「一步」或「一輪遊戲」 |

> 四個都答對才算通過；答錯一個的，現場說明後再確認一次。

---

### T2b · Maze 1D — 解釋 Q-value 更新

**任務：** 在 Q-Value Slice 折線圖（A 組）或輸出的 Q-table（B 組）上，指出某格數值並解釋是怎麼更新的。

**關鍵檢核點：**
- ✅ 通過：提到「agent 到達目標或踩雷 → 得到 reward → 那個位置的 Q 值跳動」
- ⚠️ 追問：只說「Q 值變大了」→ 問：agent 剛才做了什麼？
- ❌ 重做：指著全零的圖、或完全無法解釋

---

### T3 · Maze 2D — 在 Heatmap 上 trace 路徑

**任務：** 在 Policy Heatmap（A 組：互動熱力圖；B 組：matplotlib 圖）上，從 Start 到 Goal 描出路徑。

**關鍵檢核點：**
- ✅ 通過：能從 Start 沿箭頭方向連到 Goal（路徑合理，不需最短）
- ⚠️ 等待：Heatmap 還沒填滿（agent 跑得不夠久）→ 請學生繼續跑，稍後再驗收
- ❌ 重做：找不到 Policy Heatmap、或完全無法描述方向

---

### T4 · Heli — 描述 Training Curve 趨勢

**任務：** 跑至少 50 個 episode，描述 reward curve 趨勢。

**關鍵檢核點：**

| 學生描述 | 判斷 |
|---|---|
| 「整體往上，但有波動」 | ✅ 通過 |
| 「一直很平，沒什麼變化」 | ✅ 通過（誠實描述） |
| 「很雜亂，看不出規律」 | ✅ 通過 |
| 「越來越高然後穩定」 | ✅ 通過 |
| 只說「有變化」但說不出方向 | ⚠️ 追問：大概往上還是往下？ |
| 還沒跑到 50 ep | ❌ 請繼續跑 |

> 確認學生看的是 Episode Reward 折線圖（A 組：右側 Analysis；B 組：Colab 輸出圖），不是遊戲畫面的即時分數。

## 五、常見問題 QA

**Q: 遊戲載入失敗？**
A: 重整頁面或換用 Chrome。若整間教室同時失敗請立即回報研究者。

**Q: 學生問「Q-value 為什麼不更新」？**
A: delay 可能設太高（動畫極慢）。建議先設 delay=50，讓 agent 跑起來觀察。

**Q: 學生問「reward 一直是負的」？**
A: 正常，agent 還在探索。跑多幾個 episode 會改善。

**Q: 學生找不到 Analysis tab？（A 組）**
A: 遊戲右側側欄，點「Analysis」按鈕展開。

**Q: Colab 跑不起來？（B 組）**
A: 先確認已連接 Runtime（選單 Runtime → Connect）。若還是失敗請記錄錯誤訊息後回報。

**Q: 後測在哪裡？**
A: README 最底部「After Day 2」的連結，學生自己填。
